# 花屿 PRD V1.0

## 一、产品定位

### 产品名称
花屿（Huayu）

### 产品定位
一个帮助用户收藏共同时间、保存关系记忆的数字关系基础设施。

### 一句话描述
为那些希望未来仍然能想起的人，留下一个位置。

### 产品使命
帮助用户在共同生活结束后，依然能够保留关系存在过的痕迹。

### 产品愿景
成为数字时代的关系收藏馆（Relationship Archive）。

### 核心价值
当前主流社交产品解决的是：
- 如何认识新的人
- 如何保持沟通

花屿解决的是：
- 如何保存共同经历
- 如何降低关系衰减
- 如何让重逢变得自然

---

## 二、用户分析

### 核心用户
- **年龄**：18-30 岁
- **特征**：曾长期共享生活场景，但如今分散在不同城市的人群。

### 典型用户画像

**用户 A：毕业室友**
毕业后进入不同城市工作。没有矛盾，但联系频率明显下降。希望保留共同生活的记忆。

**用户 B：留学生**
与国内好友存在时差。关系依旧亲密，但越来越少了解彼此近况。

**用户 C：异地闺蜜**
不需要每天聊天。但希望未来关系不会消失。

**用户 D：家庭成员**
父母、子女、兄弟姐妹长期异地居住。缺少共同生活场景。

---

## 三、用户痛点分析

### Pain Point 1：共同生活结束后关系自然衰减
用户失去共同场景后，即使关系良好，也会逐渐减少互动。

### Pain Point 2：现有社交产品强化社交义务
微信、QQ 等产品强调即时回复。用户面临：回复压力、表达压力、社交负担。

### Pain Point 3：共同经历缺少长期沉淀
聊天记录不断被覆盖。朋友圈持续刷新。共同经历难以保存。

### Pain Point 4：重新联系成本高
关系沉默一段时间后，用户往往不知道如何重新开始。

### Pain Point 5：关系只能被沟通记录，无法被时间记录
用户希望保留的是共同走过的时间，而不是聊天记录本身。

---

## 四、MVP 目标

### 核心目标
验证用户是否愿意：
1. 创建共享关系空间
2. 留下生活痕迹
3. 长期保存共同时间

### MVP 原则
- 只验证核心价值
- 不追求社交活跃
- 不追求复杂玩法
- **不做社区**
- **不做聊天**

---

## 五、功能范围（MVP）

### 保留功能

#### 1. 共享花园
关系空间载体。支持：
- 双人空间
- 2~5 人小群空间
- 展示：季节、天气、昼夜变化

#### 2. 生活留痕
用户可记录：
- 一张照片
- 一句话
- 一个情绪
特点：轻量，无互动压力

#### 3. 情绪天气
用户选择：开心、平静、疲惫、难过、惊喜
系统映射：晴天、阴天、小雨、彩虹等环境变化

#### 4. 时间长卷
按时间顺序记录所有痕迹。形式类似：2026 春 → 2026 夏 → 2026 秋 → 2026 冬。帮助用户回顾共同时间。

#### 5. 关系纪念册
自动生成：第一个春天、第 100 张照片、第一年、第三个生日。形成关系资产。

#### 6. 重逢时刻
长时间未打开后再次进入。展示：*"自上次见面以来，这里又经历了一个秋天。"* 降低重连压力。

---

## 六、产品克制原则

### MVP 阶段明确不做
- 聊天
- 点赞
- 评论
- 私信
- 已读状态
- 红点提醒
- 打卡
- 连续签到
- 排行榜
- 等级体系
- 商城系统

**原因**：避免制造新的社交义务。

| **P7 我的页面** | 个人资料 / 设置 / 隐私管理 |

## 八、核心流程

### 流程 1：创建关系空间
```
登录 → 创建花园 → 生成链接 → 分享好友 → 好友加入 → 完成建园
```

### 流程 2：记录生活痕迹
```
进入花园 → 点击"留下今天" → 上传照片 / 情绪 / 一句话 → 保存 → 空间发生变化
```

### 流程 3：关系回顾
```
进入时间长卷 → 浏览季节记录 → 查看关系纪念册 → 回顾共同时间
```

### 流程 4：重逢体验
```
长时间未进入 → 再次打开 → 系统展示时间变化 → 回到空间
```

## 十二、主要风险

### 风险 1：用户价值感知不足
用户可能认为微信已经能完成相同需求。
**解决方案**：突出"关系收藏"而非"沟通"。

### 风险 2：打开频率过低
产品天然低频。
**解决方案**：强化纪念册与时间长卷价值。

### 风险 3：关系建立门槛
邀请机制可能产生压力。
**解决方案**：采用"留位机制"而非"邀请审批机制"。

### 风险 4：功能膨胀
开发过程中容易增加聊天、点赞、签到等功能，导致产品偏离定位。
**解决方案**：坚持产品克制原则。
