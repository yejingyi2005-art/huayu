## 三、数据模型

### 核心表

#### User（用户）

```sql
CREATE TABLE users (
  user_id    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nickname   VARCHAR(50) NOT NULL,
  avatar     TEXT,
  email      VARCHAR(255) UNIQUE,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| user_id | UUID | PK, default gen_random_uuid() | 用户 ID |
| nickname | VARCHAR(50) | NOT NULL | 昵称 |
| avatar | TEXT | — | 头像 URL |
| email | VARCHAR(255) | UNIQUE | 邮箱（登录用） |
| created_at | TIMESTAMPTZ | DEFAULT now() | 创建时间 |

#### Garden（关系空间）

```sql
CREATE TABLE gardens (
  garden_id  UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name       VARCHAR(100) NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now(),
  status     VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'archived')),
  invite_code VARCHAR(20) UNIQUE
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| garden_id | UUID | PK | 空间 ID |
| name | VARCHAR(100) | NOT NULL | 空间名称 |
| created_at | TIMESTAMPTZ | DEFAULT now() | 创建时间 |
| status | VARCHAR(20) | CHECK('active','archived'), DEFAULT 'active' | 状态 |
| invite_code | VARCHAR(20) | UNIQUE | 邀请码（留位机制） |

#### GardenMember（空间成员，多对多关系）

```sql
CREATE TABLE garden_members (
  garden_id UUID REFERENCES gardens(garden_id) ON DELETE CASCADE,
  user_id   UUID REFERENCES users(user_id) ON DELETE CASCADE,
  role      VARCHAR(20) DEFAULT 'member' CHECK (role IN ('owner', 'member')),
  joined_at TIMESTAMPTZ DEFAULT now(),
  PRIMARY KEY (garden_id, user_id)
);
```

#### Trace（生活痕迹）

```sql
CREATE TABLE traces (
  trace_id   UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  garden_id  UUID REFERENCES gardens(garden_id) ON DELETE CASCADE NOT NULL,
  user_id    UUID REFERENCES users(user_id) ON DELETE SET NULL,
  type       VARCHAR(20) NOT NULL CHECK (type IN ('photo', 'text', 'mood')),
  content    JSONB NOT NULL,
  -- content 结构:
  --   photo: { "url": "...", "caption": "..." }
  --   text:  { "body": "..." }
  --   mood:  { "emotion": "happy", "weather": "sunny" }
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX idx_traces_garden_created ON traces(garden_id, created_at DESC);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| trace_id | UUID | PK | 痕迹 ID |
| garden_id | UUID | FK, NOT NULL | 所属空间 |
| user_id | UUID | FK | 创建者 |
| type | VARCHAR(20) | CHECK('photo','text','mood'), NOT NULL | 类型 |
| content | JSONB | NOT NULL | 内容（结构见上） |
| created_at | TIMESTAMPTZ | DEFAULT now() | 创建时间 |

#### MemoryBook（关系纪念册）

```sql
CREATE TABLE memory_books (
  book_id    UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  garden_id  UUID REFERENCES gardens(garden_id) ON DELETE CASCADE NOT NULL,
  season     VARCHAR(20) NOT NULL,
  year       INTEGER NOT NULL,
  summary    TEXT,
  cover_url  TEXT,
  metadata   JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(garden_id, season, year)
);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| book_id | UUID | PK | 纪念册 ID |
| garden_id | UUID | FK, NOT NULL | 所属空间 |
| season | VARCHAR(20) | NOT NULL | 季节（spring/summer/autumn/winter） |
| year | INTEGER | NOT NULL | 年份 |
| summary | TEXT | — | AI 生成的摘要 |
| cover_url | TEXT | — | 封面图片 URL |
| metadata | JSONB | DEFAULT '{}' | 扩展元数据 |
| created_at | TIMESTAMPTZ | DEFAULT now() | 创建时间 |

### Supabase Row Level Security (RLS) 策略

```sql
-- Garden：只有成员可见
CREATE POLICY garden_select ON gardens FOR SELECT
  USING (garden_id IN (SELECT garden_id FROM garden_members WHERE user_id = auth.uid()));

-- Trace：只有所属空间的成员可读写
CREATE POLICY trace_select ON traces FOR SELECT
  USING (garden_id IN (SELECT garden_id FROM garden_members WHERE user_id = auth.uid()));

CREATE POLICY trace_insert ON traces FOR INSERT
  WITH CHECK (garden_id IN (SELECT garden_id FROM garden_members WHERE user_id = auth.uid()));
```

## 四、服务层约定

### 命名规范

```
src/lib/services/<entity>.service.ts
```

每个服务文件导出一个类或一组函数：

```ts
// garden.service.ts
export const gardenService = {
  async create(name: string, userId: string): Promise<Garden> { ... },
  async getById(id: string): Promise<Garden> { ... },
  async getByUser(userId: string): Promise<Garden[]> { ... },
  async join(inviteCode: string, userId: string): Promise<Garden> { ... },
  async archive(id: string): Promise<void> { ... },
}
```

### 服务层规则

1. **所有数据库操作通过服务层**，禁止在组件中直接调用 Supabase
2. **服务层函数返回领域对象**（而非原始数据库行）
3. **错误处理**：使用自定义 `AppError` 类，不抛出原始数据库错误
4. **事务**：涉及多表操作时使用 Supabase RPC 或事务
5. **缓存**：可选的 SWR/React Query 封装，由 hooks 层处理

### Hooks 层约定

```ts
// hooks/use-garden.ts
export function useGarden(gardenId: string) {
  // 封装数据获取、加载态、错误态
  return { garden, loading, error, refetch }
}
```

Hooks 命名：
- `use` + 实体名（单数表示单个，复数表示列表）
- 参数为 ID 或过滤条件
- 返回值：`{ data, loading, error, refetch }`

## 五、AI 引用机制（V2 启用）

### 架构

```
Client → Service Layer → AI Service → OpenAI API

### 文件结构

```
src/lib/ai/

### 提示词管理

所有提示词集中在 `prompts.ts`，遵循以下模板：

```ts
// prompts.ts
export const PROMPTS = {
  memorySummary: (traces: Trace[]) => `
    你正在为一个关系收藏空间生成季度回忆录摘要。
    以下是该季度 ${traces.length} 条生活痕迹：
    ${traces.map(t => `- [${t.type}] ${JSON.stringify(t.content)}`).join('\n')}

    请用温暖、诗意的中文写一段 100-200 字的摘要，
    风格类似手帐笔记，避免过于煽情。
  `,

  annualStory: (year: number, books: MemoryBook[]) => `...`,

  albumOrganizer: (photos: Trace[]) => `...`,
}
```

### 调用约定

```ts
// generators.ts
export async function generateMemorySummary(gardenId: string): Promise<string> {
  const traces = await traceService.getBySeason(gardenId, ...)
  const prompt = PROMPTS.memorySummary(traces)
  return aiClient.complete(prompt)
}
```

### 约束

- AI 生成内容**必须经过人工预览**后才可保存
- 提示词使用英文指令 + 中文输出，避免编码问题
- 所有 AI 调用增加 `timeout` 和 `retry` 机制
- API Key 仅存储在服务端环境变量中

## 六、开发约束

### 6.1 代码风格

- TypeScript 严格模式，禁止 `any`
- 使用 `const` / `let`，禁止 `var`
- 组件使用函数式组件 + Hooks
- 文件命名：`kebab-case`（如 `garden-scene.tsx`）
- 类型定义集中存放 `src/lib/types.ts`
- 禁止 `// @ts-ignore`

### 6.2 Git 提交规范

```
<type>(<scope>): <subject>

type: feat | fix | refactor | style | docs | chore | perf
scope: 组件/页面/服务名
```

示例：`feat(garden): 添加季节切换动画`

### 6.3 组件规范

- 每个组件文件不超过 200 行
- 复杂组件拆分：`index.tsx` + `components/` 子目录
- 使用 `cn()` 工具函数合并 Tailwind 类名（位于 `utils.ts`）
- 禁止内联 `style`，优先使用 Tailwind 类 + CSS 变量
- 事件处理函数命名：`handle<Event>`（如 `handleSubmit`）

### 6.4 样式约束

- 优先使用 theme.css 中的 CSS 变量（`bg-background`、`text-foreground` 等）
- 仅在装饰性组件中使用硬编码色值（如 `bg-[#F4E9C8]`）
- 动画时间统一：`duration-500`（交互）、`14s`（环境动画）
- 禁止使用 `!important`
- 所有圆角以 `--radius` 系列变量为基准

### 6.5 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| 默认 | < 768px | 单列 |
| `md:` | ≥ 768px | 双栏 grid |
| `lg:` | ≥ 1024px | 三列 / 大间距 |

### 6.6 性能约束

- 图片使用 `loading="lazy"`
- 列表渲染使用 `key`（禁止使用 index）
- 大列表使用虚拟滚动
- 页面级代码分割：`React.lazy()` + `Suspense`

### 6.7 依赖管理

- 新增依赖需在 PR 中说明用途
- 禁止引入非必要的大型库
- `package.json` 中 `dependencies` 与 `devDependencies` 严格区分

## 七、禁止破坏的逻辑

### 7.1 MVP 红线（不可触碰）

下列功能在 MVP 阶段**严禁实现**：

| 功能 | 原因 |
|------|------|
| ❌ 聊天/私信 | 与产品定位冲突，制造社交义务 |
| ❌ 点赞 | 制造社交压力 |
| ❌ 评论 | 制造社交压力 |
| ❌ 已读状态 | 制造回复压力 |
| ❌ 红点/Badge 提醒 | 对抗 Calm Technology 原则 |
| ❌ 打卡/签到 | 制造维护义务 |
| ❌ 连续签到 | 制造维护义务 |
| ❌ 排行榜 | 制造竞争压力 |
| ❌ 等级/积分体系 | 制造维护义务 |
| ❌ 商城系统 | 偏离产品定位 |
| ❌ 社区/广场 | 偏离关系收藏定位 |

### 7.2 交互红线

- **禁止**弹窗式打扰（用内联卡片或温和动画替代）
- **禁止**强制通知权限
- **禁止**未读计数（包括红点、数字角标）
- **禁止**推送催促性通知
- **禁止**加载态闪烁（使用 `min-h` 占位）

### 7.3 视觉红线

- **禁止**使用饱和色（红、亮蓝、荧光色）
- **禁止**使用灰色边框（必须用 `#D9D2C3` 暖褐色）
- **禁止**使用锐利直角（必须使用 `--radius` 系列圆角）
- **禁止**修改 CSS 变量名（theme.css 中定义的所有 `--color-*`）

### 7.4 数据红线

- **禁止**在客户端暴露 Supabase 私密 Key
- **禁止**存储用户明文密码（使用 Supabase Auth）
- **禁止**在 URL 参数中传递邀请码（必须加密或通过服务端转发）
- **禁止**删除用户数据（仅支持软删除 / 归档）

### 7.5 API 红线

- **禁止**绕过 RLS 策略（所有数据库操作必须经过 RLS）
- **禁止**在服务层以外直接调用 `supabase.from()`
- **禁止**在前端暴露 OpenAI API Key
